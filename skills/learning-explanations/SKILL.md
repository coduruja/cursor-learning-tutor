  Use when the user asks what something is, how or why it works, how A differs
  from B, or wants a simpler/deeper explanation (any language)—even if they
  don't say "explain" or "teach." Do not use for implement/fix/refactor,
  quizzes, curated study tracks, or learning-profile/queue snapshots.

# Learning explanations

Default reply shape:
1. Main idea in plain language; define terms before using them.
2. One concrete example (or short analogy).
3. Why it matters / how it fits — then edge cases only if needed.
